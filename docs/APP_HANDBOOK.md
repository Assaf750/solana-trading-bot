# APP_HANDBOOK — دليل المشغّل — محرّك نسخ تداول الأموال الذكية على سولانا

> دليلك المرجعي الوحيد لتشغيل برنامجك وفهمه. يعكس **الحالة المُتحقَّقة فعلياً** بعد بناء طبقة التطبيق الكاملة (M1–M4) — ليس وثيقة طموحات. ليس وثيقة SSOT (`docs/00`–`12` تبقى المرجع المحوكم).

**آخر تحديث:** 2026-06-11 · **المجموعة:** 1893/1893 اختبار يمرّ · **الحارسان:** أخضران

---

## 0. كيف تشغّل البرنامج (30 ثانية)

1. انقر نقراً مزدوجاً على **`START.bat`** في مجلد البرنامج.
2. يفتح المتصفح تلقائياً على **http://127.0.0.1:8787**.
3. اتبع «خطوات البدء» في صفحة Command Center.

البرنامج محلي بالكامل (localhost فقط) — لا سحابة، لا خادم خارجي، بياناتك وأسرارك على جهازك.

## 1. ما هو هذا التطبيق — وماذا يعمل الآن فعلاً

محرّك نسخ تداول كمّي على سولانا: تسجّل محافظ «أموال ذكية» رابحة، وعندما تشتري إحداها عملة، يقرّر المحرك — عبر بوّابات مخاطر صارمة — هل ينسخ الدخول وكيف يخرج.

**ما يعمل الآن (مُتحقَّق بأوامر منفَّذة):**
| القدرة | الحالة |
|---|---|
| خادم محلي + واجهة 9 صفحات تفاعلية (AR/EN) | ✅ يعمل |
| خزنة مشفّرة للأسرار (scrypt + AES-256-GCM) | ✅ يعمل |
| حفظ الإعدادات وحدود المخاطر التسعة | ✅ يعمل |
| تسجيل/متابعة المحافظ | ✅ يعمل |
| **تداول ورقي ببيانات السوق الحية** (بث mainnet + أسعار Jupiter حقيقية، أموال محاكاة) | ✅ يعمل — مُتحقَّق على mainnet |
| مفتاح إيقاف هرمي يعمل فعلاً | ✅ يعمل |
| **تداول حقيقي (REAL-LIVE)** — توقيع وإرسال فعلي على السلسلة | ✅ مبني ومُختبَر، **مقفل خلف مفتاحك أنت** |

**كيف يربح ويخسر:** يربح بالدخول مع محفظة رابحة قريباً من سعرها والخروج عند هدف الربح أو إشارة الخطر. يخسر بالدخول المتأخر (انزلاق)، أو الخروج في سيولة رقيقة، أو نسخ محفظة «ربحها وهمي». لذلك كل دخول يمرّ بفحص جدوى الخروج **قبل** الشراء، وحدود الخسارة مُلزِمة دائماً.

## 2. المعمارية (كما تعمل الآن)

```
START.bat → خادم محلي (apps/server، صفر تبعيات، 127.0.0.1:8787)
   ├─ يقدّم الواجهة (apps/operator-ui — React)
   ├─ الخزنة المشفّرة (data/vault.enc.json — مفاتيحك لا تغادر جهازك ولا تُعرض بعد إدخالها)
   ├─ الإعدادات والمحافظ والـ audit (data/*.json — ذرّية الكتابة)
   └─ المحرّك:
       بث WebSocket لمحافظك المتبوعة (mainnet مباشر)
       → كشف صفقة القائد (شراء/بيع من أرصدة المعاملة)
       → بوّابات المخاطر (مفتاح الإيقاف · حالة التشغيل · حدود Hard-Risk التسعة · الحجم · التعرّض · خسارة اليوم)
       → جدوى الخروج قبل الدخول (عرض بيع فعلي من Jupiter + سقف خسارة ذهاب-إياب)
       → التنفيذ:
           وضع paper  → تعبئة محاكاة بسعر السوق الحقيقي (دفتر ورقي)
           وضع real_live → المنفّذ الحي: سجلّ نوايا (منع تكرار) → audit قبل التوقيع
                           → توقيع Ed25519 (يرفض إن لم يكن دافع الرسوم محفظتك)
                           → إرسال → تأكيد → قراءة التعبئة الفعلية من السلسلة
       → مراقبة TP/SL كل 25 ثانية بأسعار حقيقية → خروج تلقائي
```

انقطاع البث فوق دقيقتين → **EXITS_ONLY** تلقائياً (لا دخول جديد). قفل الخزنة → توقّف المحرك. الدفتران الورقي والحقيقي **منفصلان تماماً** ولا يُخلطان أبداً.

## 3. الحزمة التقنية

JavaScript ESM (صفر تبعيات في الخادم) · React + Vite (الواجهة) · node:test (1893 اختباراً) · node:crypto (scrypt/AES-GCM/Ed25519) · Solana JSON-RPC + WebSocket (مزوّدك بالمرجع) · Jupiter Swap API (مفتاحك أو نقطة lite المجانية) · تخزين JSON ذرّي محلي في `data/` (خارج git). الحزم الـ53 الأساسية (`packages/`) تبقى أساس المراجعة والضمانات (1854 اختباراً ضمن المجموعة).

## 5. سير العمل: من الصفر إلى التداول

### المرحلة 1 — الإعداد (مرة واحدة)
1. **My Wallets & Funds:** أنشئ الخزنة بعبارة مرور قوية (8+ أحرف — احفظها؛ لا يمكن استرجاعها).
2. أدخل **رابط RPC** (مثل Helius مع مفتاحك) — يُخزَّن مشفّراً، يظهر مقنّعاً مثل `htt….com`.
3. (اختياري) مفتاح **Jupiter API** — بدونه يستخدم النقطة المجانية (أبطأ).
4. **Settings & Safety:** املأ **التسعة كلها** + `capital_limit` ثم احفظ.
5. **Wallet Intelligence:** سجّل محفظة رابحة وفعّل المتابعة.

### المرحلة 2 — التداول الورقي (ابدأ هنا دائماً)
بمجرد اكتمال المرحلة 1 والخزنة مفتوحة، يبدأ المحرك تلقائياً: يتصل بالبث، يرصد صفقات قادتك، يدخل ورقياً بأسعار حقيقية، ويخرج عند TP/SL. راقب النتائج في Trading Workspace أياماً قبل التفكير في المال الحقيقي.

### المرحلة 3 — التداول الحقيقي (قرارك أنت)
1. أنشئ **محفظة تنفيذ مخصصة** (ليست محفظتك الرئيسية) وموّلها بمبلغ تحتمل خسارته فقط.
2. **My Wallets & Funds:** استورد مفتاحها الخاص إلى الخزنة (لن يُعرض مرة أخرى أبداً) + احفظ **حدود جلسة التوقيع الأربعة** + افتح جلسة توقيع.
3. **Settings & Safety:** عندما تختفي كل الحواجز، اكتب `ACTIVATE-REAL-LIVE` حرفياً واضغط التفعيل.
4. من هذه اللحظة تُنفَّذ الصفقات الجديدة **بمال حقيقي**. زر ⛔ في Alerts يوقف كل شيء فوراً، وزر «إلغاء التفعيل» يعيدك للورقي في أي لحظة.

**نمطا النسخ:** `follow_entry_user_exit` (افتراضي آمن — يدخل معه ويخرج بهدفك) · `full_mirror` (متقدم — ينسخ بيعه أيضاً نسبياً).

## 6. كل إعداد

**Hard-Risk (مُلزِمة دائماً؛ كلها مطلوبة — أي نقص = لا تداول):** `max_daily_loss_pct/usdt` (تجاوزها يوقف الدخول ويحوّل لخروج فقط + قفل live) · `max_total_drawdown_pct` · `max_open_positions` · `max_position_size_pct` · `max_token_exposure_pct` · `max_creator/cluster/correlated_meme_exposure_pct`.
**التنفيذ:** `capital_limit` (السقف الكلي $) · `sizing_mode/value` (حجم كل صفقة، افتراضي $10).
**النسخ:** `take_profit_pct` (افتراضي 50) · `stop_loss_pct` (30 — **لا يضمن الخروج في سيولة رقيقة**) · `max_entry_slippage_vs_leader` (5).
**جلسة التوقيع (الأربعة إلزامية):** مهلة خمول · مدة قصوى · سقف قيمة موقعة بالجلسة · قفل بعد N رفض. قيم معقولة للبدء: 600000 · 3600000 · 100 · 3.

## 7. الموقّع (Signer)

الحالات: `missing` (لا مفتاح) → `locked` (مفتاح بلا جلسة/خزنة مقفلة) → `ready` (جلسة نشطة) · `degraded` (حدود الجلسة ناقصة). المفتاح يُستورد مرة واحدة، يُشفَّر، **لا يُعرض/يُسجَّل/يُصدَّر أبداً**. كل طلب توقيع يمرّ بالسلسلة: وضع حقيقي؟ مفتاح إيقاف؟ حالة تشغيل؟ خزنة؟ حدود الجلسة؟ — أي «لا» = رفض. الجلسة تُقفل تلقائياً: خمول/مدة/سقف/N رفض/مفتاح إيقاف/قفل خزنة/إطفاء البرنامج.

## 8. مفتاح الإيقاف (Kill Switch)

**⛔ زر واحد في صفحة Alerts** يوقف كل شيء فوراً: يقفل الموقّع، يمنع التوقيع/الإرسال، KILLED. **حالته محفوظة** — إعادة التشغيل لا تلغيه. الفك: اكتب `DISENGAGE` ثم استأنف (يمرّ بـ WARMING_UP). مستويات إضافية: per-mode/per-wallet/per-strategy (الخسارة اليومية في live تُفعّل قفل per-mode تلقائياً). كل تفعيل مُدقَّق.

## 9. التنبيهات والأحداث

أحداث المحرك في Trading Workspace (دخول/خروج/رفض بأسبابه الحرفية مثل `hard_risk_incomplete` و`route_invalid` و`exit_feasibility_fail`) وسجل الـ audit في Alerts. security+critical لا تُسكَت.

## 10. التقارير

الدفتر الورقي موسوم **simulated** دائماً ولا يُخلط بالحقيقي. المقياس غير المتوفر يظهر «غير متوفر» — لا يُختلق. الرسوم في الورقي تقديرية موسومة estimate؛ وفي الحقيقي تُقرأ التعبئة الفعلية من السلسلة حيث أمكن.
