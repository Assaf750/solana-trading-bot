# APP_HANDBOOK — دليل المشغّل — محرّك نسخ تداول الأموال الذكية على سولانا

> دليلك المرجعي الوحيد لفهم تطبيقك وتشغيله. مكتوب بالعربية أولاً مع المصطلحات التقنية بالإنجليزية، بلغة بسيطة، ويعكس **الحالة المُتحقَّقة الحالية فقط** — لا ميزات طموحة. يُحدَّث كلّما غيّر إصلاحٌ سلوكاً أو إعداداً أو واجهة.
>
> **ليس وثيقة SSOT** (`docs/00`–`12` هي المرجع المحوكم وغير قابلة للتعديل). هذا دليل تشغيلي.

**التاريخ:** 2026-06-11 · **الحالة:** مكتمل بنيوياً، read-only/fail-closed · **المجموعة:** 1854/1854 اختبار يمرّ · **التداول الحقيقي:** بانتظار مدخلات المالك (ليس عيب كود).

---

## 1. ما هو هذا التطبيق

محرّك تداول كمّي خاص على سولانا يفعل أمرين:
1. **نسخ المحافظ الرابحة (copy-trading):** يراقب محافظ «أموال ذكية» تختارها، وعند دخولها صفقة يقرّر — عبر بوّابات مخاطر صارمة — هل ينسخ الدخول، وكيف يخرج.
2. **صيد العملات الجديدة (new-coin hunting):** يكتشف ويُرتّب فرص التوكنات الجديدة للعرض والقرار — **لا للشراء الآلي**.

**كيف يربح أو يخسر:** يربح حين يدخل مع محفظة رابحة بسعر قريب من سعرها ويخرج عند هدف ربح أو إشارة خطر قبل الانهيار. يخسر حين يدخل متأخّراً (انزلاق)، أو حين يخرج متأخّراً في سيولة رقيقة، أو حين تكون «الربحية الظاهرة» للمحفظة المتبوعة وهمية (wash/insider). لذلك التطبيق **يخصم مخاطر الزيف ويقيس التكاليف الحقيقية** قبل أن يعدّ أي ربح حقيقياً.

**نمطان للنسخ لكل محفظة:**
- `follow_entry_user_exit` (**الافتراضي الآمن**): يدخل مع المحفظة، ويخرج بهدفك (`take_profit_pct`) أو بقواعد الأمان. بيع المحفظة الجزئي = مُعدِّل خطر لا أمر.
- `full_mirror` (**متقدّم، تفعيل صريح فقط**): يتابعها شراءً وبيعاً نسبياً؛ خروجها الكامل → خروجك الكامل بشرط الجدوى.

**عقلان:** Brain A (منحنى Pump.fun) و Brain B (PumpSwap/السوق المفتوح) مع تسليم تحكّم عند هجرة التوكن.

**القاعدة الذهبية:** عند الشكّ — يتوقّف أو يخرج فقط (Fail-Safe Not Fail-Open). اكتشاف توكن وحده **ليس إشارة شراء**.

## 2. المعمارية والتنظيم

السلسلة الإلزامية: **بيانات → إشارة → مخاطر → نيّة → مسار → توقيع → إرسال**، مُحقَّقة كحلقة وحدات نقيّة fail-closed:

```
[Data Ingestion] → [Wallet/Token Intelligence] → [Signal Engine]
        → [Risk Engine + Hard-Risk Gates] → [Intent Ledger]
        → [Route/Execution Planning] → [Transaction-Build Review]
        → [Signing Review] → [Isolated SIGN-ONLY Signer] → [Send Review]
        → [send-gate-contract: يرفض] →✗  (لا إرسال بلا مفتاح المالك)
        → [Decision Trace] (أثر كامل لكل قرار، read-only)

طبقة موازية: [Paper Execution + FIFO P&L] → [Calibration/Backtest] → [Profitability Intelligence]
طبقة العرض: [Live-Stream Boundary (read-only)] → [Operator Dashboard / operator-ui]
طبقة التفعيل: [Testnet-Send Seam] · [Mainnet Activation Seam]  → never-ready (مفتاح المالك)
```

**تخطيط المستودع:**
- `docs/` — 13 وثيقة SSOT (00–12) + هذا الدليل + التقريران التشغيليان.
- `packages/` — 53 حزمة foundation نقيّة (صفر تبعيات runtime).
- `apps/dashboard` — shell تشخيصي read-only · `apps/management-api` — API قراءة (GET فقط، محلي) · `apps/operator-ui` — واجهة 9 صفحات قابلة للنقر.
- `tools/` — حارس انحراف SSOT + حارس الآليات. `reports/` — أدلّة كل مرحلة.

**ضمانة العزل:** الإدخال الوحيد المسموح له بآلية توقيع حيّة هو `packages/isolated-signer-runtime/src/` (إدخال ALLOWLIST واحد). أي آلية حيّة خارجه يرفضها حارس الآليات.

## 5. سير العمل الكامل خطوة بخطوة

### 5.1 نمط `follow_entry_user_exit` (الافتراضي)
1. **Ingestion:** يلتقط حركة محفظة متبوعة على السلسلة.
2. **Intelligence:** يقيّم المحفظة (copyability، نوع المحفظة، مخاطر الزيف) والتوكن (جاهزية، أمان authority/extensions، تركّز).
3. **Signal:** يصنّف الحدث (`copy_event` من 17 سيناريو) — مثلاً `leader_buy`.
4. **Risk Gates (Hard-Risk):** يفحص الحدود التسعة المُلزِمة؛ أيّ تجاوز → حجب. `ev_gate_mode=warning_only` **لا** يتجاوز حدّ خسارة صلباً.
5. **Exit Feasibility:** قبل الالتزام — هل يمكن الخروج لاحقاً بمسار صالح وانزلاق مقبول؟ لا → لا دخول.
6. **Intent Ledger:** يسجّل `BUY_INTENT` بـ `intent_id` (منع تكرار/idempotency).
7. **Route → Build Review → Signing Review:** يبني الأمر ويراجعه.
8. **Isolated Signer (SIGN-ONLY):** يوقّع بمفتاح معزول لكل نداء (لا يُعرض/يُخزَّن). **يرفض إن فشل أي فحص.**
9. **Send Review → send-gate:** في البناء الحالي **يُرفض الإرسال دائماً** (لا مفتاح مالك/قرار حوكمة) — fail-closed.
10. **الخروج:** عند بلوغ `take_profit_pct` أو `stop_loss_pct` أو إشارة أمان → نيّة خروج تمرّ بنفس بوّابات الجدوى.
11. **Decision Trace + التقرير:** يُسجَّل الأثر الكامل (محاكى) في Workspace/Analytics.

### 5.2 نمط `full_mirror` (متقدّم)
كما أعلاه، لكن يتابع **بيع** القائد نسبياً أيضاً: بيع جزئي فوق `min_mirror_sell_pct` → mirror-sell؛ ما تحته يتراكم في `cumulative_ignored_sell` حتى يتجاوز العتبة. خروج القائد الكامل → خروجك الكامل بشرط الجدوى. **يتطلّب تفعيلاً صريحاً per-wallet؛ ليس default.**

## 6. كل إعداد مشروحاً

### 6.1 حدود Hard-Risk (مُلزِمة دائماً — لا يتجاوزها warning_only)
| الإعداد | معناه | إن ضُبط خطأً |
|---|---|---|
| `max_daily_loss_pct` / `max_daily_loss_usdt` | حدّ الخسارة اليومية (نسبة/قيمة) | تجاوزه → KILLED (إيقاف صلب) |
| `max_total_drawdown_pct` | حدّ الـ drawdown الكلي | غيابه/لانهائيته → التكوين غير صالح لـ REAL-LIVE |
| `max_open_positions` | أقصى مراكز متزامنة | قيمة ضخمة = تعرّض مفرط |
| `max_position_size_pct` | أقصى حجم مركز (نسبة رأس المال) | كبير = مخاطرة مركّزة |
| `max_token_exposure_pct` / `max_creator_exposure_pct` / `max_cluster_exposure_pct` / `max_correlated_meme_exposure_pct` | حدود التعرّض لتوكن/منشئ/عنقود/ميمات مترابطة | كبير = خطر ارتباط |

> **«لا لانهائية ضمنية»:** أي حقل ناقص أو `Infinity` ⇒ `real_live_config_valid:false` ⇒ غير جاهز. هذه ميزة أمان لا عطل.

### 6.2 عتبات EV (تخضع لـ `ev_gate_mode`)
`minimum_net_expectancy` · `minimum_profit_factor` · `minimum_lower_confidence_bound` · `minimum_sample_size` · `minimum_exit_success_rate` · `max_expected_drawdown_pct`. في `strict` (الافتراضي) تحجب؛ في `warning_only` تُحذّر بلا حجب (لكن لا تمسّ Hard-Risk).

### 6.3 إعدادات per-wallet
`take_profit_pct` · `stop_loss_pct` (لا يضمن الخروج في السيولة الرقيقة) · `max_time_in_position` · `max_entry_slippage_vs_leader` (تجاوزه → reject/reduce/watch-only) · `min_mirror_sell_pct` · `rebuy_cooldown` · `whipsaw_window`/`whipsaw_penalty` · `sizing_mode` (`fixed_usd`/`fixed_sol`/`pct_of_capital`) + `sizing_value`.

## 7. عمليات الموقّع (Signer)

- **الحالة في الواجهة:** `missing` · `locked` · `ready` · `degraded` · `failed`.
- **الإعداد:** لمرة واحدة، localhost فقط، استيراد المفتاح مباشرةً إلى خزنة محلية مشفّرة، **ثم لا يُعرض أبداً**. لا مفتاح في الواجهة/اللوقات/التصدير/الذاكرة المتصفّحة.
- **الفتح:** بقرار صريح منك لكل جلسة.
- **حدود الجلسة (إلزامية لتكون «ready»):** مهلة خمول قصوى · مدّة جلسة قصوى · سقف قيمة تراكمية موقّعة · قفل فوري عند Kill Switch · قفل بعد N رفض مخاطر متتالٍ. **بلا قيم صريحة غير افتراضية، الموقّع ليس جاهزاً.**
- **fail-closed:** يرفض تلقائياً إن فشل فحص مخاطر/مسار/جدوى/kill-switch. كل نداء توقيع مُدقَّق (audit).
- **الحالي:** التوقيع معزول SIGN-ONLY ومُتحقَّق؛ **الإرسال مرفوض دائماً** حتى مدخلات المالك + قرار حوكمة محوّل الإرسال.

## 8. استخدام Kill Switch

- **عالمي (Global):** يوقف كل توقيع/تنفيذ فوراً ويقفل الموقّع.
- **حسب الوضع (per-mode):** يوقف وضعاً (مثل local-live) دون المساس بالورقي.
- **حسب المحفظة (per-wallet):** يوقف نسخ محفظة بعينها.
- **حسب الاستراتيجية/العنقود.**
- **المتطلّبات:** يُبلَغ من الواجهة بنقرتين كحدّ أقصى · يعمل حتى مع تدهور RPC · حالته محفوظة (إعادة التشغيل لا تستأنف التداول صامتاً) · كل تفعيل مُدقَّق.
- **الاستئناف الآمن:** بعد إطفائه صراحةً، ويُفضَّل المرور بـ `WARMING_UP` لإعادة بناء الكاش/الصحّة.

> في البناء الحالي لا تداول حيّاً، فالـ Kill Switch هو حارس بنيوي مُتحقَّق؛ يصبح ضرورياً تشغيلياً لحظة تزويد مدخلات المالك.

## 10. التقارير والتصدير

- **المصدر:** read-model خلفي حقيقي — **لا حساب P&L داخل الواجهة**.
- **Paper دائماً موسوم simulated** ولا يُخلَط بـ real/testnet.
- **المقياس المفقود** يُعرض «غير متوفّر/insufficient_evidence» — **لا يُختلق ولا يُعرض صفراً كأنه معروف**.
- **التصدير يحجب الأسرار** (redaction إلزامي): لا مفاتيح/seed/توكنات في أي صيغة (markdown/csv/jsonl/parquet).
- **Net Business P&L** = ربح الصفقات الصافي − تكاليف التشغيل (ربح صفقات موجب ≠ ربح أعمال موجب).

## 12. مسرد المصطلحات (عربي/إنجليزي)

| المصطلح | بالعربية | المعنى |

| SIGN-ONLY | توقيع فقط | يوقّع ولا يرسل؛ الإرسال محجوب |
| never-ready seam | وصلة لا تجهز أبداً | نقطة تفعيل مثبَّتة `met:false` حتى مفتاح المالك |
| Kill Switch | مفتاح الإيقاف | إيقاف هرمي فوري للتوقيع/التنفيذ |
| EXITS_ONLY | خروج فقط | لا دخول جديد؛ الخروج مسموح |
| operating_state | حالة التشغيل | `WARMING_UP`/`ACTIVE`/`EXITS_ONLY`/`PAUSED`/`KILLED` |
